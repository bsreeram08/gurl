GraphQL

GraphQL Gurl provides first-class support for GraphQL APIs with a dedicated gurl graphql command. Basic Usage Execute a GraphQL query against an endpoint: gurl graphql "https://api.example.com/graphql" --query 'query { users { name email } }' With Variables Pass variables using --vars: gurl graphql "https://api.example.com/graphql" \ --query 'query($limit: Int!) { users(limit: $limit) { name email } }' \ --vars '{"limit": 10}' From a File For complex queries, store them in .graphql files: gurl graphql "https://api.example.com/graphql" --query-file queries/get-users.graphql Example queries/get-users.graphql: ...

gRPC, WebSocket & SSE

gRPC, WebSocket & SSE Status: gurl has internal client libraries for gRPC, WebSocket, and SSE, but these are not yet exposed as standalone CLI commands (gurl grpc, gurl ws, gurl sse do not exist). The protocol handlers are used internally by the runner and scripting engine. This page documents the intended CLI interface for a future release. The protocol implementations live in internal/protocols/ and include: gRPC — unary and streaming calls with protobuf reflection WebSocket — interactive and one-shot modes with reconnection SSE — server-sent event streaming with filtering GraphQL is the only non-HTTP protocol currently exposed as a CLI command (gurl graphql).
